场景治理

场景覆盖 API：/api/scenarios/coverage。当前薄弱项会按样本数、风险等级、模拟传感器和动态障碍模式生成推荐动作。

  • 强覆盖：干燥短草、平地、清晰边界，410 samples。
  • 薄弱覆盖：湿高草、坡地、阴影，68 samples。
  • 缺口覆盖：密集障碍物、动态宠物、不规则边界，22 samples，需要补充动态障碍仿真。

标注质检

标注 API：/api/label-schemas/current、/api/annotation-tasks、/api/annotation-tasks/{task_id}/qc。

  • 支持 LabelImg、LabelMe、CVAT，覆盖 detection、segmentation 和 V3-ready point-cloud 预留标签。
  • 检测类包括 person、pet、stone、tree、fence、obstacle-other。
  • 分割类包括 lawn、soil、boundary、forbidden-zone、obstacle-region。
  • QC 问题类型包括 missing-label、wrong-class、poor-boundary、occlusion、small-object、blur、image-quality。
  • ann-boundary-009 关联 badcase-shadow-boundary-001，用于边界 mask 复核和 Badcase 回归。

感知训练与评估

感知 API：/api/training-runs、/api/models、/api/evaluations、/api/badcases。

  • det-yolo-v2 绑定 dataset-v2，记录 learning rate、batch、epochs、mAP、recall、ONNX 和 latency 38ms。
  • seg-boundary-v1 记录 mean IoU、boundary IoU、forbidden-zone IoU、boundary deviation 和阴影边界弱点。
  • eval-perception-v2 按 validation / unseen split 展示 scenario-dense-dynamic-pet 的 small-object recall weak。
  • Badcase 可从 model version 创建，root cause 会映射为 rebalance dataset、relabel、hard-negative 或 promotion guardrail 动作。
